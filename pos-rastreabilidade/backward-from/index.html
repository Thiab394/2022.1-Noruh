    Backward-from

    1.Introdução

       A pós-rastreabilidade tem como objetivo ligar os artefatos que são criados durante o ciclo de vida de um projeto aos seus devidos requisitos elicitados. Sendo assim, neste documento iremos utilizar a técnica de rastreabilidade backward-from com o intuito de conectar os requisitos a suas respectivas técnicas de elicitação e evidenciar os elos presentes entre eles.

    2. Metodologia

       Para a análise backward-from dos requisitos iremos utilizar uma tabela dividida nas seguintes colunas:

    • Requisito
    • Ténica utilizada
    • Onde encontrar
    • Elos

       Abaixo pode-se encontrar a legenda para as abreviaturas que serão utilizadas:

    • RF: Requisito Funcional
    • RNF: Requisito Não Funcional
    • US: Histórias de Usuário
